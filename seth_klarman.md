# Seth Klarman 投資框架

## QUICK_FACTS
- **基金**：Baupost Group（1982-至今）
- **規模**：$27B
- **核心風格**：深度價值 + 極端耐心 + 高現金比例
- **最佳適用**：特殊情況、困境債、冷門資產

> **風格**：深度價值、極端耐心、高現金比例

## 核心原則
1. **安全邊際至上**：只在價格遠低於保守估值時買入；寧可錯過也不冒險。
2. **絕對回報思維**：不與指數比較；目標是「不虧錢」，而非「跑贏大盤」。
3. **耐心是武器**：可以持有大量現金數年等待機會；不怕跑輸。
4. **冷門資產**：偏好流動性差、被忽視、結構性賣壓的標的。
5. **風險 ≠ 波動**：真正風險是永久虧損；波動是創造機會的朋友。

## 投資機會分類
| 類型 | 特徵 | 例子 |
|------|------|------|
| 困境債 | 公司困境但資產價值仍在 | 破產重組債券 |
| 清算/拆分 | 市場忽視拆分後價值 | Spin-off、清算股 |
| 被迫賣壓 | 機構因規則被迫拋售 | 評級下調、指數剔除 |
| 冷門地區/行業 | 無人關注的市場 | 新興市場、小型股 |

## 快速檢查清單
- **內在價值？** 用保守假設算出的價值是多少？
- **安全邊際？** 當前價格 vs 保守價值折扣多少？（目標 30-50%）
- **為何便宜？** 是結構性賣壓還是基本面惡化？
- **下檔保護？** 最壞情況能回收多少？
- **催化劑？** 什麼事件會讓市場重新定價？
- **流動性？** 能否承受長期持有、無法退出？

## 與宏觀/週期的連結
- **恐慌/危機時期**：Baupost 最活躍的時候；現金儲備是彈藥。
- **低利率環境**：機會變少，現金比例提高；不追收益率。
- **信用週期尾聲**：困境債機會增加；但需等違約潮真正到來。
- **流動性枯竭**：非流動資產暴跌 → 超額安全邊際出現。

## 現金管理哲學
- 長期現金比例常達 30-50%
- 現金是「等待機會的期權」
- 不會為了「部署資金」而降低標準

## 經典語錄

- "Value investing is at its core the marriage of a contrarian streak and a calculator."
- "Investors should always keep in mind that the most important metric is not the returns achieved but the returns weighed against the risks incurred."

## DECISION_RULES（決策規則）

### 買入規則
```
IF 安全邊際 > 30-50%（價格 vs 保守估值）
   THEN 考慮買入
   BECAUSE 足夠的下檔保護

IF 便宜因為結構性賣壓（指數剔除、評級下調）AND 基本面未變
   THEN 深入研究
   BECAUSE 被迫賣壓創造機會

IF 困境債 AND 資產價值 > 債券價格 AND 有催化劑
   THEN 深入研究
   BECAUSE 特殊情況機會

IF 恐慌/危機時期 AND 有現金儲備
   THEN 積極尋找機會
   BECAUSE Baupost 最活躍的時候

IF 冷門資產 AND 無人關注 AND 有價值
   THEN 深入研究
   BECAUSE 流動性折價
```

### 現金管理規則
```
IF 估值普遍高估 AND 缺乏安全邊際
   THEN 提高現金比例（可達 30-50%）
   BECAUSE 等待機會的期權

IF 機會不夠好
   THEN 不部署
   BECAUSE 不為部署而降低標準

IF 危機到來 AND 有現金
   THEN 大膽部署
   BECAUSE 現金是彈藥
```

### 不買規則
```
IF 安全邊際 < 30%
   THEN 不買
   BECAUSE 保護不足

IF 便宜因為基本面惡化 AND 無改善跡象
   THEN 不買
   BECAUSE 價值陷阱

IF 下檔保護不明確
   THEN 不買
   BECAUSE 最壞情況不可接受

IF 流動性極差 AND 無法長期持有
   THEN 不買
   BECAUSE 無法承受鎖定
```

### 賣出規則
```
IF 達到內在價值
   THEN 賣出
   BECAUSE 安全邊際消失

IF 有催化劑讓市場重新定價
   THEN 等待/部分獲利
   BECAUSE 價值實現

IF 基本面惡化
   THEN 重新評估，可能賣出
   BECAUSE 論點改變
```
